# 논문 정보
- **논문 제목:**  MimicGen: A Data Generation System for Scalable Robot Learning using Human Demonstrations

- **논문 정보:** 7th Conference on Robot Learning (CoRL 2023)


# Introduction
## 연구 배경
인간의 시연 데이터로부터 학습하는 모방학습은 로봇이 복잡한 조작 행동을 습득하는 데 효과적인 패러다임으로 자리 잡았다. 최근 연구들은 대규모의 다양한 데이터셋을 통해 로봇의 일반화 능력을 비약적으로 향상 시킬 수 있음을 보여주었으나, 이러한 성공은 비용이 많이 들고 시간이 오래 걸리는 대규모 데이터 수집 과정에 크게 의존한다. 예를 들어, 간단한 물체 이동 작업 조차 적절한 성공률을 확보하기 위해 수백 개의 시연이 필요하며, 복잡한 환경으로 확장할 경우 수만 개의 데이터가 요구되어 데이터 수집의 병목 현상이 심각한 문제로 대두된다.

## 핵심 제안 방법
본 논문은 대규모 데이터셋 내에 포함된 상당 부분의 조작 행동이 서로 다른 맥락에서 반복되는 유사한 기술의 재사용이라는 점에서 주목한다. 저자들은 이러한 직관을 바탕으로, 소수의 인간 시연을 새로운 환경에 맞게 자동으로 변환하여 대규모 데이터셋을을 구축하는 MimicGen 시스템을 제안한다. MimicGen의 핵심은 시연 데이터를 Object-centric segents로 분할하고, 새로운 장면에서 변화된 물체의 pose에 맞춰 각 세그먼트를 공간적으로 변환 및 연결한다. 이 시스템은 기존의 모방학습 파이프라인에 월할하게 통합될 수 있는 범용적인 데이터 수집 시스템을 지향한다.

## 주요 성과
- **데이터 생성의 효율성 극대화**: 단 200여개의 인간 시연으로 부터 18개 작업에 대해 50,000개 이상의 시연 데이터를 자동으로 생성하는데 성공했다.

- **높은 범용성 및 정밀도**: 단순 작업부터 다중 부품 조립, 커피 준비와 같은 장기 작업 및 고정밀 작업까지 폭넓게 적용 가능함이 입증됐다.

- **실제 로봇 환경 적용**: 시뮬레이션뿐만 아니라 실제 로봇 팔에서도 데이터 생성 및 정책 학습이 가능함을 확인하여 시스템의 실용성을 증명했다.

- **경제적 효용성 입증**: 생성된 데이터가 시연 데이터와 대등한 성능을 낼 수 있음을 보여주었다.

# Problem Setup
## Imitation learning 정의
본 논문에선 로봇 조작 작업을 **MDP**(Markov Decision Process)로 가정한다. 상태공간을 행동공간으로 매핑하는 로봇 조작 정책  $\pi$ 를 학습하는 것이 목표이다.

- **데이터셋 구성**: $N$ 개의 시연 데이터 사용
$$
D = \{ (s_{i0}, a_{i0}, \dots, s_{iH_i}) \}_{i=1}^{N}
$$

- **학습 방법**: 시연에서의 행동을 그대로 따라 하도록 훈련하는 Behavioral Cloning 방식을 표준 학습 방법으로 채택했다.

## 문제정의 및 데이터 생성 목표
MimicGen의 궁극적인 목표는 소수의 인간 데이터셋을 활용하여, 동일한 작업이나 변형된 작업에 대해 대규모 데이터셋을 자동을 생성하는 것이다.

시연 데이터를 생성하는 과정은 아래와 같다.

**1.** 데이터를 생성하려는 작업의 초기 상태를 샘플링한다.
**2.** 소스 데이터셋 $D_{src}$중 하나를 선택해 새로운 장면의 물체 위치에 맞춰 궤적 $\tau^{'}$을 조정한다.
**3.** 로봇이 조정된 궤적을 실행하고, 성공한 경우에만 결과 데이터를 최종 데이터셋 $D$에 추가한다.

## 시스템 활용을 위한 3가지 핵심 가정
- **가정 1:** Delta End Effector Pose Action Space
로봇의 행동 공간은 End-Effector의 상대적인 위치/자세 변화와 그리퍼의 개폐 명렁으로 구성된다. 이를 통해 시연 데이터를 말단 장치가 따라가야할 목표 포즈의 시퀀스로 취급할 수 있게 된다.

- **가정 2:** Tasks consist of a known sequence of object-centric subtasks 
하나의 복잡한 작업은 여러개의 **물체 중심 서브태스크(Object-centric subtasks)** 가 순차적으로 이어진 것으로 가정한다.

- **가정 3:** object poses can be observed at the start of each subtask during data collection
데이터 수집 중 물체 포즈 관측 가능 데이터를 생성하는 동안에는 각 서브태스크가 시작 될 때 관련 물체의 포즈를 알 수 있다고 가정한다. 이는 시뮬레이션 환경에서는 당연한 정보이며, 실제 로봇 환경에서도 포즈 추정 알고리즘을 통해 확보할 수 있는 정보이다.

# Method
MimicGen의 핵심은 소수의 인간 시연 데이터를 물체 중심의 서브태스크 단위로 나누고, 이를 새로운 환경의 물체 위치에 맞춰 공간적으로 변환하여 재사용하는 것이다.

전체 과정은 크게 세 단계로 나뉜다.

##  Parsing the Source Dataset into Object-Centric Segments
먼저, 소스 데이터셋 $D_{src}$에 포함된 모든 시연 궤적 $\tau$를 Sub-task 단위의 $\tau_{i}$로 나눈다.

- **분할기준:** 각 작업이 알려진 물체 중심 Sub-task의 시퀀스로 구성된다는 가정하에, 각 sub-task가 끝나는 지점을 자동으로 감지하여 시연을 분할한다.
- **결과:** 하나의 시연은 "머그컵 잡기", "머그컵 이동하기" 등과 같이 특정 물체와 관련된 연속적인 세그먼트들의 집합이 된다.

## Transforming Source Data Segments for a New Scene
새로운 장면(물체의 위치나 자세가 바뀐 환경)이 주어지면, 각 서브태스크에 대해 다음 과정을 거친다.

- **Chossing a reference segment:** 소스 데이터셋에서 해당 Sub-task에 해당하는 세그먼트 중 하나를 무작위 또는 유사도 기반(Nearest-Neighbor)으로 선택한다.

- **Transforming the source subtask segment:** 선택된 인간의 움직임을 새로운 물체 포즈 $T^{C^{'}}_{W}$에 맞춰 변환한다. 핵심 아이디어는 로봇 말단 장치와 물체 사이의 상대적인 위치관계를 시연Eo와 동일하게 유지하는 것이다.
$$
T^{C^{'}_t}_{W} = T^{O_{0}}_{W}(T^{O^{'}_0}_{W})^{-1}T^{C^{'}_t}_{W}
$$

## Interpolation & Execution
변환된 세그먼트들을 롭소이 부드럽게 따라갈 수 있도록 연결하고 실행한다.

- **Interpolation:** 로봇의 현재 위치와 변ㄹ환된 세그먼트의 시작 지점 사이가 멀 수 있으므로, 두 지점을 부드럽게 연결하는 직선 경로를 추가한다.

- **노이즈 추가 실행:** 변환된 궤적을 따라 로봇이 움직일때, 약간의 가우시안 노이즈를 행동에 추가한다. 이는 자웅에 학습할 정책이 작은 오차에도 견딜 수 있는 강건함을 갖추게한다.

## Success Filtering
Mimicgen은 생성된 모든 데이터를 무조건 사용하지 않는다.
- 로봇이 변환된 궤적을 끝까지 실행한 후, 작업의 성공 여부를 확인한다.
- 오직 성공한 시도에서 나온 상태와 행동의 시퀀스만을 최종 데이터셋에 추가하여 데이터의 품질을 보장한다.

# Experiment Setup
연구진은 MimicGen이 다양한 환경과 작업에서 얼마나 효과적으로 데이터를 생성하고 로봇을 학습시킬 수 있는지 확인하기 위해 광범위한 실험을 설계했다.

## Task & Variants
실험은 총 18개의 작업에 대해 수행되었으며, 이는 크게 다섯 가지 카테고리로 분류된다
- **기초 작업 (Basic):** 상자 쌓기 등

- **접촉 중심 작업 (Contact-Rich):**: 너트 끼우기, 바늘 꿰기 등 정밀한 접촉이 필요한 작업

- **장기 작업 (Long-Horizon):** 커피 준비, 주방 정리 등 여러 단계의 행동을 연결해야 하는 작업

- **모바일 매니퓰레이션 (Mobile Manipulation):** 로봇 팔뿐만 아니라 베이스(바닥) 이동이 동시에 필요한 작업

- **고정밀 조립 (Factory):** 1mm 오차 수준의 정밀도가 요구되는 산업용 조립 작업

각 작업은 물체의 초기 위치 분포에 따라 세 가지 난이도로 분류된다.
- **$D_0$ (Default):** 인간 시연 데이터가 수집된 기본 환경
- **$D_1$ (Broad):** 물체의 위치와 회전 변화가 더 넓은 분포를 가짐
- **$D_2$ (Challenging):** 물체의 위치가 서로 바뀌는 등 데이터 생성 및 학습이 매우 어려운 환경

## 데이터 수집 및 생성 방식
- **소스 데이터셋:** 각 작업당 단 10~25개의 인간 시연 데이터만을 사용했다
- **자동 생성 데이터셋:** MimicGen을 사용하여 각 작업 변형 $(D_0, D_1, D_2)$ 마다 1,000개의 성공적인 시연 데이터를 자동으로 생성했다. 만약 로봇이 실행 도중 실패하면 해당 시도는 버리고, 성공한 데이터가 1,000개가 될 때까지 반복했다.

## Methodology
- **학습 알고리즘:** 시연 데이터를 모방하는 BC-RNN (Behavioral Cloning with RNN) 방식을 표준으로 사용했다

- **관측 공간 (Observation Spaces):**
    - Image: 전면 카메라와 로봇 손목 카메라의 RGB 입력을 직접 사용 (84x84 해상도)

    - Low-dim: 물체의 실제 위치(Ground-truth) 정보를 사용

- **데이터 증강:** 이미지 기반 에이전트의 경우, Pixel Shift 무작위화를 적용하여 강건함을 높였다.

## Real Robot Evaluation
시뮬레이션에서의 성공을 넘어 실제 하드웨어인 Panda 로봇 팔에서도 실험을 진행했다.

- **하드웨어:** RealSense D415(전면) 및 D435(손목) 카메라를 사용하여 물체의 포즈를 추정했다

- **데이터 생성:** 실제 환경에서 10개의 소스 데이터를 기반으로 약 100~200개의 데이터를 성공적으로 생성하고 정책을 학습시켰다

# Experiments
연구진은 MimicGen이 생성한 데이터가 로봇의 학습에 얼마나 실질적인 도움이 되는지, 그리고 다양한 변수에 대해 얼마나 유연하게 대응하는 지 광범위한 실험을 통해 검증했다.

## 데이터 확장을 통한 성능의 비약적 향상
단순히 소수의 인간 시연 데이터(Source)만으로 학습했을 때와 MimicGen을 통해 확장된 대규모 데이터셋으로 학습했을 때의 성능 차이는 극명하게 나타났다. 

- **성공률의 대폭 상승:** 'Square' 작업의 경우 성공률이 11.3%에서 90.7%로, 'Threading' 작업은 19.3%에서 98.0%로 향상되었다.
- **복잡한 작업 수행:** 인간의 데이터만으로는 학습하기 어려웠던 정밀 조립(Factory) 작업이나 장기(Long-horizon) 작업에서도 MimicGen 데이터로 학습한 에이전트는 뛰어난 성능을 보였다
.
## 환경 및 하드웨어의 한계를 넘는 확장성 (Transfer)
MimicGen은 단순히 같은 환경에서 데이터를 복제하는 것을 넘어, 한 번도 본 적 없는 새로운 환경으로의 확장이 가능하다는 점을 입증했다.
- **Object Transfer:** 소스 데이터에는 없던 새로운 디자인의 머그컵이나 12가지의 서로 다른 머그컵에 대해서도 성공적으로 데이터를 생성하고 학습에 활용할 수 있었다.

- **Robot Transfer:** 가장 흥미로운 점 중 하나는 Panda 로봇에서 수집된 시연 데이터를 바탕으로 Sawyer, IIWA, UR5e와 같은 다른 로봇 하드웨어를 위한 데이터를 생성할 수 있었다는 것이다. 각 로봇의 제어 특성이 다름에도 불구하고 학습된 정책은 모든 로봇에서 80~91%의 유사한 성능을 나타냈다.

- **Mobile Manipulation:** 고정된 팔뿐만 아니라 베이스 이동이 포함된 'Mobile Kitchen' 작업에서도 성공적으로 데이터를 확장하여 성능을 2.0%에서 46.7%까지 끌어올렸다 
.
## MimicGen 데이터 vs. 실제 인간 시연 데이터
연구진은 "MimicGen이 만든 가짜 데이터가 진짜 인간의 데이터만큼 좋을까?"라는 질문에 답하기 위해 비교 실험을 진행했다.
- **대등한 효율성:** 단 10개의 소스 데이터를 MimicGen으로 200개로 늘린 결과가, 실제 인간이 200번 직접 시연하여 얻은 데이터와 대등한 성능을 냈다.
- **경제적 이점:** 이는 인간 작업자가 수백 번의 시연을 반복하는 고된 노동 없이도, 소수의 데이터만 있다면 MimicGen을 통해 경제적으로 학습 데이터를 무한히 확장할 수 있음을 의미한다

## 실제 로봇 적용 결과 (Real Robot Evaluation)
시뮬레이션을 넘어 실제 Panda 로봇 팔에서도 검증을 진행했다.
- 10개의 소스 시연만으로 훨씬 넓은 작업 영역 $D_1$에 대한 데이터를 생성했다.
- 기존 소스 데이터만으로 학습한 에이전트는 성공률이 0%였던 어려운 환경에서도, MimicGen 기반 에이전트는 유의미한 성공률을 기록하며 실제 환경에서의 실용성을 증명했다.

## 심층 분석 (Insights)
- **생성 성공률과 모델 성능의 관계:** 데이터 생성 과정에서의 성공률(Data Generation Rate)이 낮더라도(예: 8.2%), 결과적으로 성공한 데이터들만 모아 학습시키면 모델의 성능은 매우 높게(예: 76.0%) 나타날 수 있다. 이는 재생(Replay) 방식을 직접 정책으로 쓰는 것보다, 이를 데이터 수집 도구로 활용해 모델을 훈련시키는 것이 훨씬 유리함을 보여준다.

- **데이터 양의 임계점:** 데이터셋의 크기를 200개에서 1000개로 늘릴 때는 성능 향상이 뚜렷했으나, 1000개에서 5000개로 늘릴 때는 향상 폭이 줄어드는 수확 체감(Diminishing returns) 현상이 관찰되었다.

# Limitations
MimicGen은 혁신적인 시스템이지만, 실제 적용 시 고려해야 할 몇 가지 한계점이 존재한다.
- **서브태스크 정보 및 포즈 추정 의존성:** 작업이 물체 중심의 서브태스크 시퀀스로 구성된다는 사전 지식이 필요하며, 데이터 생성 시 각 단계에서 물체의 포즈 정보가 정확히 제공되어야 한다.

- **단순한 데이터 필터링:** 생성된 데이터의 품질을 '성공 여부'만으로 판단하기 때문에, 데이터셋에 편향(Bias)이 생기거나 로봇의 비정상적인 움직임(Artifacts)이 포함될 가능성이 있다.

- **충돌 회피 미보장:** 인간의 시연 세그먼트를 잇는 선형 보간(Linear interpolation) 방식은 주변 환경의 기하학적 구조를 고려하지 않으므로, 의도치 않은 충돌이 발생할 수 있다.

- **작업 및 물체의 제약:** 현재 시스템은 강체(Rigid objects)를 다루는 준정적(Quasi-static) 작업에 최적화되어 있으며, 변형 가능한 물체나 매우 동적인 움직임이 필요한 작업에는 적용하기 어렵다.

- **단일 로봇 팔 구성:** 현재는 단일 로봇 팔 작업에 대해서만 검증되었으며, 양팔 로봇(Multi-arm) 작업으로의 확장은 향후 연구 과제이다

# Conclusion
본 연구에서 제안한 MimicGen은 소수의 인간 시연을 재구성하여 로봇 학습을 위한 대규모 데이터셋을 구축하는 강력하고 경제적인 시스템이다.

결론적으로 MimicGen은 로봇 학습 분야에서 데이터 중심의 관점을 강화하며, 인간의 개입을 최소화화면서도, 고품질 데이터를 확보할 수 있는 실질ㅈ거인 경로를 제시했다는 점에서 큰 의의가 있다.
