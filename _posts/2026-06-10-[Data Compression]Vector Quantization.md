## Vector Quantization
벡터 양자화는 스칼라 양자화와 달리, 입력데이터를 개별적으로 처리하지 ㅇ낳고 연속된 데이터의 묶음으로 만들어 한꺼번에 처리하는 방식이다.
- **기본원리:** $a\times b$ 크기의 블록을 하나의 $L$차원 벡터로 간주한다.

* **구성요소**
    * **Code book:** 대표 벡터들의 짛합
    * **Code-Vector/Code word:** Code book에 저장된 각각의 대표 벡터
    * **Index:** 각 코드 벡터에 부여된 이진 번호

## Vector Quantization 작동 과정
벡터 양자화는 압축 과정은 복잡하지만, 복원 과정은 매우 간단하다는 특징이 있다.

- **Encoding:** 입력 벡터와 Code book 내의 모든 코드 벡터를 비교하여 Euclidean Distance가 가장 까까운 Code-Vector를 찾는다. 그 후 해당 Code-Vector의 인덱스만 출력한다

- **Decoding:** 수신한 인덱스를 가지고 코드북에서 해당 벡터를 찾아내는 Table look up 방식을 이용한다.

## VQ vs SQ
동일한 bit rate에서 VQ가 SQ보다 항상 더 낮은 Distortion을 보인다.

1. **상관관계의 활용:** 인접한 데이터간의 높은 상관관계를 이용하여 코드 벡터의 수를 최소화 할 수 있다.

2. **유연성:** 블록 크기와 코드북 크기를 조절할 수 있어 설계가 자유롭습니다.

3. **세밀한 영역 분할:** SQ는 양자화 영역이 직사각형 형태에 국한되지만, VQ는 고차원 공간에서 육각형 등 다양한 형태의 Cell로 영역을 나누어 소스 분포에 더 최적화 할 수 있다.

4. **샤논의 정리:** 데이터가 서로 독립이더라도 여러 개를 묶어 처리하는 것이 개별 처리보다 유리하거나 최소한 동등한 성능을 낸다.

## Codebook 설계와 LBG 알고리즘
VQ의 성능 Code Book을 얼마나 잘 설계하느냐에 달려 있습니다. 가장 대표적인 설계 방식이 LBG 알고리즘(Linde-Buzo-Gray algorithm)이다.

### K-means 알고리즘(기초)
데이터를 $k$개의 그룹으로 묶는 클러스터링 기법입니다.
1. 초기 대표 패턴을 설정한다.
2. 각 데이터를 가장 가까운 대표 패턴에 할당한다.
3. 할당된 데이터들의 평균값을 계산하여 대표 패턴을 업데이트 한다.
4. 변화가 없르떄까지 반복한다.

### LBG 알고리즘(확장)
K-means 알고리즘에 Splitting 개념을 더한 것이다.
1. 전체 훈련 데이터의 평균을 구해 하나의 코드 벡터로 시작한다.
2. 분할: 기존 코드 벡터에 미세한 변화($\epsilon$)
3. 늘어난 코드 벡터들을 가지고 K-Means 과정을 거쳐 최적화한다.
4. 블록이 커질수록 성능은 좋아지지만 계산 복잡도가 기하급수적으로 증가한다.
