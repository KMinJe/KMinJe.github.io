## Transform Coding 개요 및 동기
- **정의 및 목적:** 데이터 시퀀스를 Reversible mapping을 통해 변환하여, 최대한 적은 수의 성분에 대부분의 정보를 집중 시키는 기법이다.

* **주요 동기:**
    * 입력 데이터를 더 깊이 있게 해석할 수 있다.
    * 변환 영역에서 데이터 간의 상관관계가 제거 된다.
    * 데이터를 특성에 따라 여러 범주로 분해하여 처리할 수 있다.

* **에너지 압축 원리:** 신호의 변동성이 클수록 정보량이 많다. 변환을 통해 특정 계수에 큰 분산을 몰아주고, 정보가 적은 나머지 계수들을 버림으로써 압축들 달성한다.

## Transform Coding의 기본 3단계
1. Transform: 데이터를 일정 크기의 블록($N$)으로 나누고 가역적 선형 변환을 수행한다.
2. Quantization: 변환된 계수들을 원하는 비트율과 왜곡 정도에 맞춰 양자화 한다.
3. Entropy Coding: 양자화된 데이터를 Huffman, AC, RLC 등을 사용하여 이진코드로 변환한다.

## 주요 변환 기법들

### KLT(Karhuen-Leove Transform)
- **특징:** 입력 신호의 자기 상관 행렬의 고유벡터를 기저로 사용하는 최적의 변환이다
- **장점:** 에너지 압축 효율을 최대화 하며, 변환 영역에서 계수간 상관간계를 완벽히 제거한다.
- **단점:** 데이터 통계에 의존적이라어서 매번 새로 계산해야 하므로 계산 복잡도가 높고 비실용적이다.

### DCT(Discrete Cosine Transform)
- **특징:** DFT를 기반으로 입력 데이터를 대칭 확장하여 계산한다. 멀티 미디어 압축에서 가장 널리 사용된다.
- **장점:** 복소수가 아닌 실수 연산만 필요하여 DFT보다 단순하며, 데이터 의존성이 없으면서도 KLT에 근접하는 우수한 에너지 압축 성능을 보인다.
- **구성:** 저주파 성분을 나타내는 DC계쑤와 고주파 성분을 나타내는 AC 계수들로 구성된다.

### 그외 변환 방법들
- **DST:** 상관계수가 낮을 떄 KLT에 근접하는 성능을 보이며 DCT를 보완한다.
- **Haar Transform:** 가장 간단한 형태의 변화으로, 기저 벡터가 1과 -1로만 구성된다.
- **DWHT:** 계산이 매우 단순하며, 사인파를 사용하지 않는 변환 중 가장 효율적이다.

## Bit Allocation 전략
변환 계수마다 담고 있는 정보량이 다르므로, 각기 다른 비트를 할당해야한다.

- **최적 비트 할당:** Lagrange Multipliers를 사용하여 주어진 총 비트 내에서 전체 왜곡을 최소화하도록 할당한다.

- **구역 샘플링:** 계수의 평균적 특성에 따라 특정 구역에 비트를 몰아주고 나머지는 버린다. JPEG의 양자화 테이블이 이러한 방식이다.

- **Threshold Coding:** 특정 임계값보다 큰 계수만 전송한다. 위치 정보와 값을 함꼐 보내며 Zigzag Scan을 주로 사용한다.


## JPEG 압축 프로세스
1. **전처리:** RGB 컬러 공간을 $YC_BC_R$로 변환하고 채도 성분을 서브 샘플링을 한다.
2. **Level Shifting:** 8비트 픽셀 값에서 128을 빼서 [-128, 127]범위로 맞춘다.
3. **8x8 블록 DCT:** 영상을 8x8 블록으로 나누어 DCT를 수행한다.
4. **양자화:** 양자화 테이블을 사용하여 계수를 나눈다. 저주파는 정밀하게, 고주파는 거칠게 양자화하여 많은 고주파 계수가 0이 된다.
5. **Entropy Coding:**
    * DC 계수: 이전 블록과의 차이값을 허프만 코딩한다
    * AC 계수: 지그재그 스캔으로 1차우너 배열을 만든 후, 0의 개수를 세는 RLC와 허프만 코딩을 결합하여 압축한다.
